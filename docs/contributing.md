# Contributing Guide

## Overview

Thank you for your interest in contributing to the Art Gallery Documentation project.

Contributions that improve the application, documentation, accessibility, or overall user experience are welcome.

---

# Ways to Contribute

You can contribute by:

- Fixing documentation errors
- Improving writing clarity
- Correcting spelling or grammar
- Updating screenshots
- Improving accessibility
- Refactoring code
- Fixing bugs
- Adding new features
- Improving responsive design

---

# Development Workflow

1. Fork the repository.
2. Create a new feature branch.

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes.
4. Test the application.
5. Commit your work.
6. Push your branch.
7. Open a Pull Request.

---

# Documentation Standards

When writing documentation:

- Use clear and concise language.
- Write in Markdown.
- Use descriptive headings.
- Keep sections focused on a single topic.
- Include code blocks where appropriate.
- Verify all instructions before publishing.

---

# Code Style

When modifying the project:

### HTML

- Use semantic HTML elements.
- Maintain consistent indentation.

### CSS

- Reuse existing CSS variables.
- Keep selectors organized.
- Avoid duplicate styles.

### JavaScript

- Write descriptive function names.
- Keep functions focused on one responsibility.
- Comment complex logic when necessary.

---

# Commit Message Convention

Use descriptive commit messages.

Examples:

```text
docs: update installation guide
docs: improve user guide
fix: resolve mobile navigation issue
feat: add gallery filtering
style: improve responsive layout
```

---

# Pull Request Checklist

Before submitting a Pull Request, ensure that:

- Documentation is accurate.
- Markdown formatting is correct.
- Links work correctly.
- Code has been tested.
- Screenshots (if added) are up to date.
- No unnecessary files are included.

---

# Reporting Issues

If you find a bug or documentation issue, include:

- A clear description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser and operating system (if relevant)

---

# Code of Conduct

Please be respectful and constructive when providing feedback or reviewing contributions.

The goal is to create a welcoming environment for all contributors.
